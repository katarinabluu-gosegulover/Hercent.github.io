## Git 명령어 학습하기 (메인)

커밋 명령어: git commit

---

브랜치: "하나의 커밋과 그 부모 커밋들을 포함하는 작업 내역"
아래의 명령으로 새 브랜치 생성

```bash
git branch [브랜치명]
```

아래의 명령으로 새 브랜치로 이동가능.

```bash
git checkout [브랜치명]
```

이렇게 하면 변경분을 커밋하기 전에 새 브랜치로 이동함.

---

특정 브랜치를 main 브랜치와 합치기
아래의 명령어로 수행 가능.

```bash
git merge [브랜치명]
```

---

리베이스는 기본적으로 커밋들을 모아서 복사한 뒤, 다른 곳에 떨궈 놓는 것.

리베이스를 하면 커밋들의 흐름을 보기 좋게 한 줄로 만들 수 있다는 장점이 있고, 
리베이스를 쓰면 저장소의 커밋 로그와 이력이 한결 깨끗해짐.
아래의 명령어로 main에 리베이스 가능. (**특정 branch에 있는 상태**)

```bash
git rebase main
```

---

HEAD: 현재 작업중인 커밋
HEAD는 항상 작업트리의 가장 최근 커밋을 가리키고, 작업트리에 변화를 주는 git 명령어들은 
대부분 HEAD를 변경하는것으로 시작함.
일반적으로 HEAD는 브랜치의 이름을 가리키고있습니다(bugFix와 같은)
커밋을 하게 되면, bugFix의 상태가 바뀌고 이 변경은 HEAD를 통해서 확인이 가능함.

HEAD 분리 하기는 checkout을 통하여 가능하고,  (원래 브랜치에 붙어 있다가 commit으로 이동)
전에 사용한것 처럼 새 브랜치로 이동한다거나 등을 할 수 있음.
---
상대 참조

항상 해시값을 직접 입력하기에는 한계가 있음.

한번에 한 커밋 위로 움직이는 `^`
한번에 여러 커밋 위로 올라가는 `~<num>`

---

브랜치 강제로 옮기기

 -f 옵션을 이용해서 브랜치를 특정 커밋에 직접적으로 재지정 할 수 있음.
예를 들면:
```bash
git branch -f main HEAD~3
```
(강제로) main 브랜치를 HEAD에서 세번 뒤로 옮김.

---

Git에서 작업 되돌리기

Git에서 변경한 내용을 되돌리는 방법은 크게 두가지가 있음.  하나는 `git reset`을 쓰는거고, 다른 하나는 `git revert`를 사용하는 것이다.
각자의 컴퓨터에서 작업하는 로컬 브랜치의 경우 리셋(reset)을 잘 쓸 수 있습니다만, "히스토리를 고쳐쓴다"는 점 때문에 다른 사람이 작업하는 리모트 브랜치에는 쓸 수 없다는 한계가 존재한다.

변경분을 되돌리고, 이 되돌린 내용을 다른 사람들과 공유하기 위해서는, `git revert`를 써야함.
리버트를 하면 다른 사람들에게도 변경 내역을 push 할 수 있습니다.

---

Git 체리-픽 (Cherry-pick)
이 시리즈의 첫 명령어는 `git cherry-pick` 입니다. 다음 과 같은 형태로 사용:

`git cherry-pick <Commit1> <Commit2> <...>` (특정 브랜치에서 main으로 복사본을 보내고 싶은 순서대로)
현재 위치(HEAD) 아래에 있는 일련의 커밋들에대한 복사본을 만들겠다는 것을 간단히 줄인 말이다.

---

Git 인터렉티브 리베이스(Interactive Rebase)

원하는 커밋을 모를때 사용한다.

인터렉티브 리베이스는 rebase 명령어를 사용할 때 -i 옵션을 같이 사용한다.

이 옵션을 추가하면, git은 리베이스의 목적지가 되는 곳 아래에 복사될 커밋들을 보여주는 UI를 띄울것이다. 각 커밋을 구분할 수 있는 각각의 해시들과 메시지도 보여준다.

---

Git 태그
프로젝트에서 중요한 지점들에 영구적으로 표시를 할 방법은 git 태그를 이용하는 것이다.
중요한 점은, Git 태그는 커밋들이 추가적으로 생성되어도 절대 움직이지 않는다는 것이다.

```bash
git tag v1 main
```
커밋 지정 하지 않으면 HEAD가 가리키는 HEAD에 태그 생성됨.

---

Git Describe
git에는  가장 가까운 태그에 비해 상대적으로 어디에 위치해있는지 *describe(묘사)*해주는 명령어가

Git describe 는 다음의 형태를 가지고 있음:

```bash
git describe <ref>
```

<ref>에는 commit을 의미하는 그 어떤것이던 쓸 수 있고, 만약 ref를 특정 지어주지 않으면, git은 그냥 지금 체크아웃된곳을 사용한다 (HEAD).

명령어의 출력은 다음과 같은 형태로 나타납니다:

```text
<tag>-<numCommits>-g<hash>
```

tag는 가장 가까운 부모 태그를 나타낸다. numCommits은 그 태그가 몇 커밋 멀리있는지를 나타낸다. <hash>는 묘사하고있는 커밋의 해시를 나타낸다.
