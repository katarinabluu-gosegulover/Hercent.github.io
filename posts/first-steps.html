이 블로그를 왜 만들었는지

빠르게 흘러가는 메모를 오래 남는 기록으로 바꾸고 싶어서 이 공간을 만들었다. GitHub Pages는 복잡한 세팅 없이 정적 파일만으로도 바로 공개할 수 있어서 시작점으로 적절했다.

작게 시작하기

처음부터 완벽한 블로그를 만들려 하면 금방 멈춘다. 그래서 이번에는 메인 페이지, 소개 페이지, 글 목록, 글 두 개만 먼저 만들어 두고 이후에 필요한 부분을 추가하는 구조로 잡았다.

무엇을 기록할지

앞으로는 프로젝트를 진행하면서 생기는 시행착오, 새로 배운 개념, 작업 중 떠오른 아이디어를 짧은 글 형태로 남길 생각이다. 길고 정제된 글보다 자주 쌓이는 기록이 더 중요하다.

다음 단계

이 샘플 글을 지우고 실제 첫 글을 올리면 된다. 제목과 날짜를 바꾸고, 같은 폴더에 새 HTML 파일을 추가한 뒤 `posts/index.html` 에 링크만 연결하면 블로그가 바로 확장된다.
